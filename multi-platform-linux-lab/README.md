## Overview

This project documents the deployment, configuration, testing, and troubleshooting of Linux environments across multiple platforms.

The objective was to gain hands-on experience with Linux administration in physical, virtualized, mobile, and Windows-integrated environments while comparing deployment methods, platform limitations, and management approaches.

---

## Objectives

* Deploy Linux across multiple platforms
* Compare Linux deployment methods
* Practice Linux system administration
* Configure graphical Linux environments
* Troubleshoot installation and configuration issues
* Evaluate platform-specific limitations
* Develop Linux command-line proficiency
* Gain experience with virtualization and compatibility layers

## Lessons Learned

* Linux can be deployed across a wide range of platforms using different methods.
* Root access significantly impacts Linux deployment flexibility on Android devices.
* Virtualization provides a safe environment for experimentation and learning.
* WSL offers strong integration between Windows and Linux workflows.
* Troubleshooting skills are critical when working across multiple platforms.
* Different deployment methods involve different trade-offs in performance, usability, and functionality.
